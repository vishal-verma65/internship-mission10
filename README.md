# Data Hub API

A RESTful API built with Node.js, Express, and MongoDB Atlas. Features full CRUD operations, relational modeling with Mongoose ODM, `.populate()` for relational data hydration, and aggregation queries.

---

## Tech Stack

- **Runtime** — Node.js (ES Modules)
- **Framework** — Express
- **Database** — MongoDB Atlas (M0 Sandbox)
- **ODM** — Mongoose
- **Logging** — Winston
- **Environment** — dotenv

---

## Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/your-username/data-hub.git
cd data-hub
```

### 2. Install dependencies

```bash
npm install
```

### 3. Configure environment variables

```bash
cp .env.example .env
```

Open `.env` and fill in your values:

```env
PORT=3000
MONGO_URI=mongodb+srv://<username>:<password>@cluster0.xxxxx.mongodb.net/data-hub?retryWrites=true&w=majority
JWT_SECRET=your_jwt_secret_here
```

### 4. Start the server

```bash
# Development (auto-restarts on file change)
npm run dev

# Production
npm start
```

Server runs at `http://localhost:3000`.

## API Endpoints

### Health Check

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/` | API status check |

### Users

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/users` | Create a new user |
| GET | `/users` | Get all users |
| GET | `/users/:id` | Get user by ID |
| DELETE | `/users/:id` | Delete user by ID |

**Request body for `POST /users`:**
```json
{
  "name": "Vishal Verma",
  "email": "vishal@test.com"
}
```

### Posts

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/posts` | Create a new post |
| GET | `/posts` | Get all posts (with author populated) |
| GET | `/posts/:id` | Get post by ID (with author populated) |
| DELETE | `/posts/:id` | Delete post by ID |
| GET | `/posts/top/recent` | Get top 3 most recent posts |

**Request body for `POST /posts`:**
```json
{
  "title": "My First Post",
  "content": "This is the content of my first post.",
  "authorId": "<userId>"
}
```

## Data Models

### User

| Field | Type | Required |
|-------|------|----------|
| `name` | String | Yes |
| `email` | String (unique) | Yes |
| `createdAt` | Date | Auto |
| `updatedAt` | Date | Auto |

### Post

| Field | Type | Required |
|-------|------|----------|
| `title` | String | Yes |
| `content` | String | Yes |
| `createdAt` | Date | Auto |
| `authorId` | ObjectId (ref: User) | No |

## Logging

Winston logs every incoming request to both the console (colorized) and `logs/requests.log` (plain text). The `logs/` directory is created automatically on first run.
